# 产品图片 AI 识别系统 PRD

## 目标

将现有产品 6 面图 Excel 模板库开发成可部署的 Web 系统。用户可以上传单张产品图片，或上传包含多张图片的文件夹，由 AI 持续识别并返回产品编码、产品名称、瓶/听/箱、图片视角和匹配候选。

## 用户

- 业务人员：上传图片，查看产品识别结果和候选模板。
- 管理/运维人员：维护模板库、重建索引、配置模型、检查任务状态。

## 第一版范围

### 单张识别

- 支持上传 JPG、PNG、WebP 图片。
- 展示上传图片预览。
- 返回最佳识别结果：
  - 产品编码
  - 产品名称
  - 瓶/听/箱
  - 视角
  - 相似度
  - 命中模板图片
- 返回 Top 5 候选结果。
- 低置信度时提示人工复核。

### 文件夹批量识别

- 前端支持选择文件夹上传。
- 自动过滤支持的图片格式。
- 创建后台识别任务。
- 页面关闭后任务继续运行。
- 支持查看任务进度。
- 每张图片独立记录状态，单张失败不影响整批任务。
- 支持导出 CSV / Excel 结果。

### 模板库

- 支持读取现有 Excel 中的 `DISPIMG` 单元格图片。
- 抽取产品编码、产品名称、瓶/听/箱、图片视角和模板图片。
- 生成 `manifest.json`、`manifest.csv` 和模板图片目录。
- 使用向量模型生成模板向量索引。

### 模型配置

- 默认向量模型：`qwen3-vl-embedding`。
- 默认多模态模型：`qwen3-vl-plus`。
- 前端可修改：
  - 向量模型名
  - 多模态复核模型名
  - 是否启用多模态复核
  - 低置信度阈值
- API Key 只保存于服务器环境变量，不展示在前端。
- 修改向量模型后提示重建索引。

### 部署

- 部署到公司华润雪花服务器。
- 使用 nginx 反向代理。
- 后端服务由 systemd 管理。
- 前端静态资源由 nginx 托管。
- `.env` 仅保存在服务器，不提交版本记录。

## 暂不包含

- 用户登录和复杂权限。
- 多租户。
- 在线编辑产品信息。
- 人工纠错回写 Excel。
- 分布式任务队列。
- 大型向量数据库。
- 云对象存储。
- 数据库审计日志。

## 页面

### 识别工作台

- 模板库状态摘要。
- 单张图片上传区。
- 文件夹上传入口。
- 上传图片预览。
- 最佳识别结果。
- 命中模板图。
- Top 5 候选列表。

### 批量任务

- 批量任务列表。
- 任务状态：等待中、识别中、已完成、失败、部分失败。
- 总数、已完成数、失败数。
- 任务结果表。
- 导出按钮。

### 模板库状态

- 产品记录数。
- 模板图片数。
- 索引状态。
- 最近构建时间。
- 模板来源 Excel 文件列表。

### 模型设置

- 当前向量模型名。
- 当前多模态模型名。
- 低置信度阈值。
- 多模态复核开关。
- 保存配置。
- 恢复默认配置。

## 验收标准

### 模板库

- 能从两份 Excel 抽取不少于 112 个产品记录。
- 能抽取不少于 731 张模板图片。
- 每张模板图片都能关联到产品编码、产品名称、瓶/听/箱和视角字段。
- 重建模板库后生成 `manifest.json` 和向量索引文件。

### 单张识别

- 上传模板库中的原图，Top 1 必须命中自身产品和视角。
- 上传相似实拍图，返回 Top 5 候选和相似度分数。
- 图片格式不合法时返回明确错误。
- 未配置模型 Key 时返回明确错误。

### 批量识别

- 用户可以选择包含多张图片的文件夹。
- 上传后立即生成任务 ID。
- 后端持续处理任务，页面关闭后不丢失。
- 前端能查看任务进度。
- 单张失败不会中断整个任务。
- 任务完成后可导出结果。

### 前端

- 上传中有加载状态。
- 成功后展示最佳结果和候选列表。
- 失败时展示可读错误，不出现空白页。
- 低置信度结果提示人工复核。

### 部署

- 服务器上可通过指定域名或 IP 访问页面。
- 服务重启后自动恢复。
- `.env` 不提交到代码仓库。
- nginx 支持常见图片上传大小。
